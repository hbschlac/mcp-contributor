A Claude Code skill for contributing **upstream to the Model Context Protocol (MCP)** governance org — the spec, official SDKs, and docs at [github.com/modelcontextprotocol](https://github.com/modelcontextprotocol).

**Not for** building MCP servers. Use the `mcp-builder` skill (separate) for that.

## What it covers

- Triage (small change vs SEP) — when to open a PR vs follow the SEP process
- Environment setup + prerequisites (Node 24+, npm 11+, Git, language tooling)
- Fork → branch → PR workflow for the spec repo and 10 official SDKs
- Schema changes (`npm run generate:schema`) + docs via MDX + Mintlify components
- Full SEP lifecycle — types, statuses, prototype rules, sponsor-finding, Core Maintainer meetings
- Governance model — 6-rung contributor ladder, decision delegation, escalation matrix, succession
- Working Groups + Interest Groups — active groups, charter template, participation tiers
- Communication etiquette — Discord vs Discussions vs Issues, security-issue handling
- Roadmap-aligned priorities to boost SEP acceptance odds
- Licensing, trademark, antitrust (LF Projects policies)
- Protocol primer — participants, primitives, transports, lifecycle

## Install

```bash
cd ~/.claude/skills/
git clone https://github.com/hbschlac/mcp-contributor.git
```

Then trigger in any Claude Code session with any of:

- "contribute to MCP"
- "write an SEP"
- "MCP spec PR"
- "MCP good first issue"
- "find an MCP maintainer"
- "I want to do an SEP"
- "/mcp-contributor" (explicit invocation)

Full trigger list is in `SKILL.md`'s frontmatter description.

## Keep it fresh

MCP is actively evolving. Content in `SKILL.md` is synthesized from public MCP docs and will rot.

```bash
./refresh.sh              # audit all tracked sources; report drift + new pages
./refresh.sh --quiet      # only output if action needed (cron-friendly)
```

Outputs:
- `hashes.json` — current SHA-256 of every covered URL
- `refresh-report.md` — human-readable drift + gap report
- Exit code 1 if drift or new pages detected (0 = clean)

Dependencies: `curl`, `yq`, `sha256sum` (or `shasum` on macOS). Install `yq` via `brew install yq`.

**Recommended cadence:** weekly. Schedule via cron, launchd, or any task scheduler.

## How it's structured

| File | Purpose |
|---|---|
| `SKILL.md` | The skill itself — frontmatter + instructions loaded when triggered |
| `sources.yml` | Map of every source URL → coverage status + anchor in SKILL.md |
| `refresh.sh` | Drift detector + new-page detector (runs against `sources.yml` + `llms.txt`) |
| `hashes.json` | Baseline hashes of covered pages (regenerated by `refresh.sh`) |
| `refresh-report.md` | Last refresh run output |
| `README.md` | This file |

## Coverage status

See `sources.yml` for the full map. Coverage tiers:

- `covered` — synthesized into SKILL.md
- `gap-high` — contributor-critical, not yet ingested
- `gap-med` — useful context, not yet ingested
- `gap-low` — reference-only, SKILL.md links out
- `sep-ref` — individual SEPs; link-only unless central to workflow

## Contributing to this skill

Improvements welcome via PR. If you find a claim in `SKILL.md` that's wrong or stale:
1. Open an issue with the source URL showing the correct current state
2. Or submit a PR that updates the content AND bumps `fetched` in `sources.yml`

If MCP publishes a new page that belongs in the skill, `refresh.sh` will flag it on its next run. Triage it into `sources.yml`.

## Why this skill exists

MCP is maturing from a single-vendor project into LF-governed infrastructure. Contributing upstream now has a real bar: SEP process, sponsor-finding, WG coordination, proper prototype requirements. This skill compresses the scattered governance docs into one operational playbook so Claude can drive real contributions without the user having to re-read eight community pages each time.

Built as a working example of the "LLM-ingestible governance doc" pattern. If your OSS project is adding governance structure, `sources.yml` + `refresh.sh` is a reusable template — just swap the URLs.
